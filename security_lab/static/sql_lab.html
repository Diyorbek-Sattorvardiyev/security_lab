SQL Injection Laboratoriyasi

SQL in'eksiyasi yordamida zaif kirish formalaridan foydalaning, so'ngra parametrlashtirilgan so'rovlar hujumni qanday to'xtatishini ko'ring.

SQL INJECTION QANDAY ISHLAYDI

Foydalanuvchi kiritgan ma'lumot bevosita SQL so'rovlariga qo'shilganda, hujumchi SQL kodni in'eksiya qila oladi. Misol: SELECT * FROM users WHERE username='' OR '1'='1'--' — bu har doim rost qaytaradi va autentifikatsiyani butunlay chetlab o'tadi.

ZAIF Kirish

Satr birlashtirish

Foydalanuvchi nomi maydonida ushbu yuklanmalarni sinab ko'ring:

' OR '1'='1'-- admin'-- ' OR 1=1--

Avtomatik to'ldirish uchun bosing

NATIJA

YARATILGAN SO'ROV:

SELECT * FROM users WHERE username='?' AND password='?'

XAVFSIZ Kirish

Parametrlashtirilgan

Bir xil yuklanmalar bu yerda BLOKLANADI:

Parametrlashtirilgan so'rovlar kiritilgan ma'lumotni MA'LUMOT sifatida ko'rib chiqadi, hech qachon SQL kodi sifatida emas. Ma'lumotlar bazasi drayveri barcha maxsus belgilarni avtomatik ravishda qochiradi.

NATIJA

XAVFSIZ SO'ROV:

SELECT * FROM users WHERE username=? AND password=?
-- Kiritma parametr sifatida BOG'LANGAN, SQL sifatida tahlil qilinmaydi

Kod Taqqoslash

ZAIF KOD 

# XAVFLI - Ishlatmang!
query = f"""SELECT * FROM users
  WHERE username='{username}'
  AND password='{password}'"""

cursor.execute(query)

XAVFSIZ KOD 

# XAVFSIZ - Parametrlashtirilgan so'rov
query = """SELECT * FROM users
  WHERE username=?
  AND password=?"""

cursor.execute(query,
  (username, password))
